# Sprints

Each file in this directory is a sprint log: what was scoped, what was
actually built, what was deferred, and the decisions worth remembering.

Format: `NN-short-title.md`. One sprint per file. Don't rewrite history when
priorities change — open a new sprint.

## Index

- [01-scaffold.md](01-scaffold.md) — initial scaffold: docker-compose, Postgres schema, Rust API, Vite/React web app reading from `/bootstrap`.
- [02-interactions.md](02-interactions.md) — interactions: drag-to-schedule, draft modal, switchable user tabs, week navigation, editable estimates / status / subtasks, done-task visuals.
- [03-auth-and-projects.md](03-auth-and-projects.md) — Google OAuth, sessions, per-user data scoping, login screen, project create.
- [04-polish.md](04-polish.md) — Lucide icons, palette, project edit, calendar layout reshape (kill left rail, project filter into right rail), inline subtask edit, alignment fixes.
- [05-sync.md](05-sync.md) — Closed the local-first round-trip: outbox-driven `POST /ops` for push, `processed_ops` change log + `GET /changes?since=N` for pull, `applyRemoteOp` runs the same handlers as local mutations.
- [06-bootstrap-fix-and-dev-seeder.md](06-bootstrap-fix-and-dev-seeder.md) — Quick cleanup: fix a `/bootstrap` 500 on empty op log, share seeder logic with a new `POST /auth/dev-seed` endpoint, and add a "Seed dev data" button to the login screen.
- [07-collab-and-issue-links.md](07-collab-and-issue-links.md) — First real collab: editable project membership with soft-delete, teammates' calendars render via bootstrap blocks-in-scope, manual issue links to Jira/Notion via per-project URL templates, rail Mine/All toggle so you can log time on others' tasks.
- [08-workspaces.md](08-workspaces.md) — Multi-tenancy: `workspaces` + `workspace_members` as the company boundary, personal workspace auto-created per user, `X-Workspace-Id` header on every scoped route, two role axes (workspace + project), topbar workspace switcher, custom `<Select>` everywhere so popovers escape `overflow: auto` containers, sync pill rolled up offline + pending + retry/discard, persisted snapshot for offline boot.
- [09-deploy.md](09-deploy.md) — Production shape for the first Fly.io deploy at `usefira.app`: API serves the SPA same-origin, dev endpoints behind a cargo feature flag, multi-stage Dockerfile, `fly.toml` skeleton.
- [10-websockets.md](10-websockets.md) — Server-push for the change feed: Postgres `pg_notify` → per-process `PgListener` → in-process fan-out → `/api/ws` nudge socket. Drops the 2 s pull cadence to a 60 s fallback while keeping the outbox push hot.
- [11-playground-and-polish.md](11-playground-and-polish.md) — In-browser "Try as Maya" playground (no backend, persisted snapshot, every network call gated on `playgroundMode`); typography pass via a single `--fs-scale` knob, Lucide sidebar icons, login wordmark refit, inbox row recompose.
- [12-snapshot-and-subtask-polish.md](12-snapshot-and-subtask-polish.md) — Unified the playground seed with `seed.rs` via a build-time `dump-bootstrap` Rust bin so both share one source of truth; subtask drag-reorder; Lucide icons replace the last CSS glyphs; modal label tone softened.
- [13-deletions-and-user-channel.md](13-deletions-and-user-channel.md) — Destructive flows (task / project / workspace delete with shared `ConfirmDelete` + type-to-confirm); sync hygiene (server-rejected ops block the queue with a recoverable retry/discard popover, drained-queue auto-`hydrate()`); a dedicated per-user WS channel for events that decide *who* can subscribe to the workspace feed — fixes a chicken-and-egg where a new member never received the op that added them.
- [14-account-linking-and-personal-overlay.md](14-account-linking-and-personal-overlay.md) — Two opt-in read-only overlays so a person sees one timeline: account linking (type a partner's email, paired identity chip in the topbar, mutual calendar visibility) and personal-workspace overlay (project your personal blocks onto a team workspace). Sticky receive-side modal driven from the persisted row so it survives refresh. Fixed an infinite-loop selector that returned fresh array literals.
- [15-project-roles-and-mobile-polish.md](15-project-roles-and-mobile-polish.md) — Per-project role grew to {`owner`, `lead`, `member`, `inactive`} with passive roles hidden from inbox assignee groups; inbox gets an Unassigned bucket and `setTaskAssignee(null)` auto-flips Now → Later; calendar rail sorts by inbox order; first round of mobile polish (`<Select>` opens on iOS, scroll fixes, ghost scrollbar gone).
- [16-mobile-touch-drag.md](16-mobile-touch-drag.md) — Touch drag end-to-end via the hybrid pattern: pointer events for state, document-level non-passive `touchmove` for `preventDefault`-suppressed scroll. 220 ms long-press anywhere on a row, plus the explicit `::` grip. Calendar block tap-to-reveal-then-tap-to-open. iOS preventDefault-mid-gesture turned out to need touch events specifically, not pointer events — that took most of the sprint to land.
- [17-mobile-layout.md](17-mobile-layout.md) — Real mobile layout (not "desktop, made tappable"): 3-day calendar centered on today with single-day prev/next, slide-over sidebar with hamburger, topbar pruned to breadcrumb + sync pill + Log out, inbox row stripped to grip · check · title, task rail stacked under the calendar on phones. Independent `dayOffset` cursor so desktop and mobile don't fight when you switch.
- [18-mobile-touch-mk2.md](18-mobile-touch-mk2.md) — Mobile layout met touch reality. Dynamic viewport (`100dvh`) so the inbox / calendar / modal stop hiding under Safari toolbars. Long-press visual fires *only* at lock — earlier flash made quick taps look like rejected gestures. Two-tap grace window for time-block action buttons so the first tap can't accidentally fire X / tick / duplicate. Big debug saga that ended with a structural fix: the inbox grip's parallel pointer handlers were silently intercepting touches and bypassing the long-press hook entirely. Drag paths now split cleanly by input type — Desktop = HTML5, Mobile = long-press, single path. Plus PWA install (icons, manifest, safe-area insets) and the new shared `useLongPress` hook.
- [19-email-invites.md](19-email-invites.md) — Email-based workspace invites replace the global user-search picker, which was an onboarding wall for anyone not already in the system. Owner enters an email, recipient gets a sticky modal mirroring the account-link receive UX, accept switches them straight into the just-joined workspace. Membership editing is now one-mutation-per-click — Save Changes is title-only, role changes fire immediately, removal is a typed-email confirm hitting a dedicated `DELETE` endpoint that cascades the soft-delete into project memberships. Tasks and time blocks owned by ex-members stay (it's history, not stale data). Plus the iCloud "Hide My Email" QuickType chip finally went away after switching the inbox add-task field from `<input>` to `<textarea rows={1}>` — the heuristic is keyed off the element type, nothing else worked.
- [20-time-block-polish.md](20-time-block-polish.md) — Time-block action row scales with `--fs-scale` (Lucide icons sized via a CSS class, not the prop) and collapses to a kebab + above-block popup on short / overlapping blocks. Touch finally separates "single-press scrolls" from "long-press drags" via `touch-action: pan-y` + an always-on `touchmove` listener that branches on a freeze flag set at long-press lock. Tap activation deferred to release so a swipe-to-scroll doesn't highlight every block it passes. Task modal grows full block CRUD: click-to-edit date / start / end (duration is read-only computed); trash button to delete; "+ Add block" creates a 1 h block ending at the next 15-min boundary; end-time `00:00` is the special-case "end-of-day" save (next day's 00:00). Plus: real-time clock + today highlight via a 60-s `setClockTick`, modal backdrop close gated on mousedown-AND-click both landing on the backdrop (so text-selection drag-out doesn't dismiss), Enter-on-subtask creates the next subtask via a new `addSubtask(taskId, title, afterId?)` path with a real fractional-indexing midpoint sort_key. Plus the SQL recipe for moving a project across workspaces.
- [21-tags.md](21-tags.md) — Tags grow up: per-project entity with UUID identity, color, edit/rename/recolor/delete in `ProjectModal`, multi-select picker on the task modal, sticky AND/OR filter strip on the inbox. Five new ops (`tag.create`, `tag.set_title`, `tag.set_color`, `tag.delete`, `task.set_tags`); `tasks.tags TEXT[]` dropped, `task_tags` join table replaces it; `task.create.tag_ids` replaces `task.create.tags`. Filter persists through `partialize`, prunes phantoms on bootstrap and on `tag.delete`, auto-attaches the active filter to quick-add tasks so they don't immediately disappear from the row they were typed into. Inbox row trail caps at 3 chips (1 on phones) + `+N` overflow, sorted match-first under an active filter and length-desc inside the filter strip itself. Faint color-mix outline replaces the colored dot. Mobile places the Tags editor in the main pane (sidebar is closed by default on phones). `--done` desaturated from Tailwind green-700 to a muted sage so the ticked time block stops shouting.

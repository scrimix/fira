# Sprints

Each file in this directory is a sprint log: what was scoped, what was
actually built, what was deferred, and the decisions worth remembering.

Format: `NN-short-title.md`. One sprint per file. Don't rewrite history when
priorities change ‚Äî open a new sprint.

## Index

- [01-scaffold.md](01-scaffold.md) ‚Äî initial scaffold: docker-compose, Postgres schema, Rust API, Vite/React web app reading from `/bootstrap`.
- [02-interactions.md](02-interactions.md) ‚Äî interactions: drag-to-schedule, draft modal, switchable user tabs, week navigation, editable estimates / status / subtasks, done-task visuals.
- [03-auth-and-projects.md](03-auth-and-projects.md) ‚Äî Google OAuth, sessions, per-user data scoping, login screen, project create.
- [04-polish.md](04-polish.md) ‚Äî Lucide icons, palette, project edit, calendar layout reshape (kill left rail, project filter into right rail), inline subtask edit, alignment fixes.
- [05-sync.md](05-sync.md) ‚Äî Closed the local-first round-trip: outbox-driven `POST /ops` for push, `processed_ops` change log + `GET /changes?since=N` for pull, `applyRemoteOp` runs the same handlers as local mutations.
- [06-bootstrap-fix-and-dev-seeder.md](06-bootstrap-fix-and-dev-seeder.md) ‚Äî Quick cleanup: fix a `/bootstrap` 500 on empty op log, share seeder logic with a new `POST /auth/dev-seed` endpoint, and add a "Seed dev data" button to the login screen.
- [07-collab-and-issue-links.md](07-collab-and-issue-links.md) ‚Äî First real collab: editable project membership with soft-delete, teammates' calendars render via bootstrap blocks-in-scope, manual issue links to Jira/Notion via per-project URL templates, rail Mine/All toggle so you can log time on others' tasks.
- [08-workspaces.md](08-workspaces.md) ‚Äî Multi-tenancy: `workspaces` + `workspace_members` as the company boundary, personal workspace auto-created per user, `X-Workspace-Id` header on every scoped route, two role axes (workspace + project), topbar workspace switcher, custom `<Select>` everywhere so popovers escape `overflow: auto` containers, sync pill rolled up offline + pending + retry/discard, persisted snapshot for offline boot.
- [09-deploy.md](09-deploy.md) ‚Äî Production shape for the first Fly.io deploy at `usefira.app`: API serves the SPA same-origin, dev endpoints behind a cargo feature flag, multi-stage Dockerfile, `fly.toml` skeleton.
- [10-websockets.md](10-websockets.md) ‚Äî Server-push for the change feed: Postgres `pg_notify` ‚Üí per-process `PgListener` ‚Üí in-process fan-out ‚Üí `/api/ws` nudge socket. Drops the 2 s pull cadence to a 60 s fallback while keeping the outbox push hot.
- [11-playground-and-polish.md](11-playground-and-polish.md) ‚Äî In-browser "Try as Maya" playground (no backend, persisted snapshot, every network call gated on `playgroundMode`); typography pass via a single `--fs-scale` knob, Lucide sidebar icons, login wordmark refit, inbox row recompose.
- [12-snapshot-and-subtask-polish.md](12-snapshot-and-subtask-polish.md) ‚Äî Unified the playground seed with `seed.rs` via a build-time `dump-bootstrap` Rust bin so both share one source of truth; subtask drag-reorder; Lucide icons replace the last CSS glyphs; modal label tone softened.
- [13-deletions-and-user-channel.md](13-deletions-and-user-channel.md) ‚Äî Destructive flows (task / project / workspace delete with shared `ConfirmDelete` + type-to-confirm); sync hygiene (server-rejected ops block the queue with a recoverable retry/discard popover, drained-queue auto-`hydrate()`); a dedicated per-user WS channel for events that decide *who* can subscribe to the workspace feed ‚Äî fixes a chicken-and-egg where a new member never received the op that added them.
- [14-account-linking-and-personal-overlay.md](14-account-linking-and-personal-overlay.md) ‚Äî Two opt-in read-only overlays so a person sees one timeline: account linking (type a partner's email, paired identity chip in the topbar, mutual calendar visibility) and personal-workspace overlay (project your personal blocks onto a team workspace). Sticky receive-side modal driven from the persisted row so it survives refresh. Fixed an infinite-loop selector that returned fresh array literals.
- [15-project-roles-and-mobile-polish.md](15-project-roles-and-mobile-polish.md) ‚Äî Per-project role grew to {`owner`, `lead`, `member`, `inactive`} with passive roles hidden from inbox assignee groups; inbox gets an Unassigned bucket and `setTaskAssignee(null)` auto-flips Now ‚Üí Later; calendar rail sorts by inbox order; first round of mobile polish (`<Select>` opens on iOS, scroll fixes, ghost scrollbar gone).
- [16-mobile-touch-drag.md](16-mobile-touch-drag.md) ‚Äî Touch drag end-to-end via the hybrid pattern: pointer events for state, document-level non-passive `touchmove` for `preventDefault`-suppressed scroll. 220 ms long-press anywhere on a row, plus the explicit `::` grip. Calendar block tap-to-reveal-then-tap-to-open. iOS preventDefault-mid-gesture turned out to need touch events specifically, not pointer events ‚Äî that took most of the sprint to land.
- [17-mobile-layout.md](17-mobile-layout.md) ‚Äî Real mobile layout (not "desktop, made tappable"): 3-day calendar centered on today with single-day prev/next, slide-over sidebar with hamburger, topbar pruned to breadcrumb + sync pill + Log out, inbox row stripped to grip ¬∑ check ¬∑ title, task rail stacked under the calendar on phones. Independent `dayOffset` cursor so desktop and mobile don't fight when you switch.
- [18-mobile-touch-mk2.md](18-mobile-touch-mk2.md) ‚Äî Mobile layout met touch reality. Dynamic viewport (`100dvh`) so the inbox / calendar / modal stop hiding under Safari toolbars. Long-press visual fires *only* at lock ‚Äî earlier flash made quick taps look like rejected gestures. Two-tap grace window for time-block action buttons so the first tap can't accidentally fire X / tick / duplicate. Big debug saga that ended with a structural fix: the inbox grip's parallel pointer handlers were silently intercepting touches and bypassing the long-press hook entirely. Drag paths now split cleanly by input type ‚Äî Desktop = HTML5, Mobile = long-press, single path. Plus PWA install (icons, manifest, safe-area insets) and the new shared `useLongPress` hook.
- [19-email-invites.md](19-email-invites.md) ‚Äî Email-based workspace invites replace the global user-search picker, which was an onboarding wall for anyone not already in the system. Owner enters an email, recipient gets a sticky modal mirroring the account-link receive UX, accept switches them straight into the just-joined workspace. Membership editing is now one-mutation-per-click ‚Äî Save Changes is title-only, role changes fire immediately, removal is a typed-email confirm hitting a dedicated `DELETE` endpoint that cascades the soft-delete into project memberships. Tasks and time blocks owned by ex-members stay (it's history, not stale data). Plus the iCloud "Hide My Email" QuickType chip finally went away after switching the inbox add-task field from `<input>` to `<textarea rows={1}>` ‚Äî the heuristic is keyed off the element type, nothing else worked.

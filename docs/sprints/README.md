# Sprints

Each file in this directory is a sprint log: what was scoped, what was
actually built, what was deferred, and the decisions worth remembering.

Format: `NN-short-title.md`. One sprint per file. Don't rewrite history when
priorities change — open a new sprint.

## Index

- [01-scaffold.md](01-scaffold.md) — initial scaffold: docker-compose, Postgres schema, Rust API, Vite/React web app reading from `/bootstrap`.
- [02-interactions.md](02-interactions.md) — interactions: drag-to-schedule, draft modal, switchable user tabs, week navigation, editable estimates / status / subtasks, done-task visuals.
- [03-auth-and-projects.md](03-auth-and-projects.md) — Google OAuth, sessions, per-user data scoping, login screen, project create.
- [04-polish.md](04-polish.md) — Lucide icons, palette, project edit, calendar layout reshape (kill left rail, project filter into right rail), inline subtask edit, alignment fixes.
- [05-sync.md](05-sync.md) — push (`POST /ops` + outbox worker + status pill) and pull (`processed_ops` as change log + `GET /changes?since=N` + cursor + applyRemoteOp) — local-first round-trip closed.
- [06-bootstrap-fix-and-dev-seeder.md](06-bootstrap-fix-and-dev-seeder.md) — fix `/bootstrap` 500 on empty `processed_ops` (`MAX(seq)` NULL decode), share seeder logic between CLI bin and a new `POST /auth/dev-seed` endpoint, add "Seed dev data & sign in" button to the login screen behind `DEV_AUTH=1`.
- [07-collab-and-issue-links.md](07-collab-and-issue-links.md) — editable project membership (split into `project.update` + `project.set_members` ops with soft-delete on `project_members`), bootstrap blocks-in-scope so teammates' calendars render, rail Mine/All toggle + title filter so `time_blocks.user_id` becomes a usable column, manual issue links via `external_url_template` + `task.set_external_id`, inbox `(you)` no longer hardcoded to Maya, `fmtMin` sign fix.
- [08-workspaces.md](08-workspaces.md) — `workspaces` + `workspace_members` as the company-level tenant boundary; personal workspace auto-created per user (Google OAuth + seeder); `X-Workspace-Id` header on all scoped routes; **two role axes** — workspace `owner`/`member` and project `lead`/`member` — gating workspace and project mutations; topbar workspace switcher; owner-only settings modal in the sidebar; custom `<Select>` replaces every native dropdown so popovers stay inside the editorial palette and escape `overflow: auto` containers; sync pill now combines `Offline · N pending`, surfaces a failed-ops popover with retry/discard, and suppresses fast-tick flicker; `zustand/middleware/persist` snapshot lets the app boot offline from cache when `/me` can't reach the server.
- [09-deploy.md](09-deploy.md) — production-shape changes for the first Fly.io deploy at `usefira.app`: api serves the SPA same-origin via `ServeDir` (CORS dropped), dev-mode endpoints behind a `dev_auth` cargo feature, `API_BIND_ADDR=0.0.0.0:8080`, multi-stage Dockerfile, `fly.toml` skeleton.
- [10-websockets.md](10-websockets.md) — server-push for the change feed via `pg_notify` + per-process `PgListener` + in-process `Hub` fan-out + `GET /api/ws` nudge socket. 2 s `/changes` poll dropped to 60 s fallback; `syncOutbox` kept on 2 s. Local multi-instance test rig (`pnpm dev:second`, second API on `:3001`) verifies cross-machine nudges.
- [11-playground-and-polish.md](11-playground-and-polish.md) — in-browser "Try as Maya" playground (no backend, JS-baked seed, persisted snapshot, `playgroundMode` gates every network call); typography/iconography polish (single `--fs-scale` knob, Lucide sidebar icons, brand-mark size fix, login wordmark refit, seeded project icons aligned to the picker registry, inbox row recompose, copy-as-markdown affordance on the task modal).
- [12-snapshot-and-subtask-polish.md](12-snapshot-and-subtask-polish.md) — playground seed unified with `seed.rs` via a build-time `dump-bootstrap` Rust bin emitting a frozen snapshot JSON (lib refactor, shared `applyBootstrap`, `setFrozenNow` on `time.ts`, wallclock-independent block state); subtask drag-reorder (new `subtask.reorder` op + DnD UI), Lucide icons replace CSS tick / `×` / `+`, modal labels relaxed from wide-tracked mono to subtler mono uppercase, topbar `Playground` chip joined to the SyncPill.
- [13-deletions-and-user-channel.md](13-deletions-and-user-channel.md) — destructive flows (task / project / workspace delete with shared `ConfirmDelete` + type-to-confirm guard for the high-blast-radius cases); sync hygiene (server-rejected ops stop auto-retrying and now block the queue, surfaced via toasts that carry the server's actual error, drained-queue auto-`hydrate()` after the user finishes resolving failed ops); a dedicated per-user WS channel (`/api/ws/user` + `pg_notify('user_changes')` + `Hub::notify_user`) for membership / role / workspace events that decide *who* can subscribe to the workspace-scoped feed — fixes the chicken-and-egg where new members never received the op that added them; migration 0010 drops `processed_ops` FKs to projects/workspaces so the op log survives entity deletion.
- [14-account-linking-and-personal-overlay.md](14-account-linking-and-personal-overlay.md) — account linking via email (`user_links` table with canonicalized pair + partial unique on accepted, `POST /api/links { email }` looks up by email rather than workspace-scoped picker, partner nudges ride the user channel cross-workspace), sticky received-request modal driven from the persisted row so it survives refresh / new tabs, paired identity chip in the topbar (two `topbar-me` boxes + lucide `Link` between them) when accepted; personal-workspace overlay (`/api/personal/calendar`) reusing the linked overlay's `LinkedTask` projection — second toolbar toggle, distinct left-stripe styling so both overlays can coexist; bug fix for a `useSyncExternalStore` infinite loop caused by selectors returning fresh object literals.
- [15-project-roles-and-mobile-polish.md](15-project-roles-and-mobile-polish.md) — per-project role axis grew to {`owner`, `lead`, `member`, `inactive`} (migrations 0012/0013): `owner` and `inactive` are passive — hidden from inbox assignee groups unless tasks are assigned — workspace owners auto-get `owner` on every project and can de/up-rank themselves, `set_project_members_tx` re-anchored on workspace ownership instead of `projects.owner_id`, `has_project_lead_authority` accepts both `owner` and `lead`; inbox gets an Unassigned bucket and `setTaskAssignee(null)` auto-flips Now → Later; calendar rail sorts by inbox order and "All" widens to Now+Later; draft modal stops pre-selecting a project (was easy to mis-target); mobile Select fix (`pointerdown` + `touch-action: manipulation` so dropdowns open on iOS); section-heading consolidation in the task modal, description ghost-scrollbar fix, and `×`/`✓` glyphs replaced with Lucide everywhere except the inbox task-rail done check.
- [16-mobile-touch-drag.md](16-mobile-touch-drag.md) — touch drag end-to-end via the hybrid pattern (pointer events for state, document-level non-passive `touchmove` for `preventDefault`-suppressed scroll): inbox task + subtask reorder via `::` grip and 220 ms long-press, time block move/resize, rail-task → calendar scheduling for tablets; assignee-group drop detection added to touch path so empty groups still highlight cyan and reassign; calendar block tap-to-reveal-then-tap-to-open via `data-active` and `pointerType`/`wasActive` on `DragState`; inbox subtasks behave as part of the parent row (tap opens task, long-press drags task) — inline edit moved to the modal; time block row in task modal gets a responsive `grid-template-areas` that drops the avatar and stacks date/time on phone widths; viewport tuned to `width=device-width, initial-scale=0.5` and `.cal-rail` hidden below 1000 px; smaller fixes — cyan-stuck-after-drop, description ghost scrollbar, iOS text-selection during drag, persistent grip on touch.
- [17-mobile-layout.md](17-mobile-layout.md) — real mobile layout (not "desktop, made tappable"): 3-day calendar centered on today with single-day prev/next stepping (independent `dayOffset` cursor + `dayAnchorFor` / `dayOfWeekLabelFor` helpers, `gridAnchor` rename in CalendarView), slide-over sidebar with `Menu` hamburger replacing the Fira crumb on phones, topbar pruned to breadcrumb + sync pill + Log out (avatars, link button, week title, playground pill all hidden), cal-toolbar pruned to nav + totals (user-picker + link toggles hidden), inbox row stripped to grip · check · title (tags / ext-id / "Xh over" / "no est" all removed), inbox-doc padding cut to `8 px` at phone widths so the title runs near the edge, project-header meta hidden at ≤700 px; task rail back on phones — stacked under the calendar with long-press drag matching the inbox; viewport locked at `initial-scale=1, user-scalable=no` with inputs floored at `16 px` to defeat iOS focus-zoom; `.cal-grid-wrap` opts out of text-selection so long-press on the grid no longer selects gutter labels; iCloud "Hide My Email" suppressed on inbox inputs via `name`/`autoComplete="off"`/`type="text"` trio; new shared `useIsMobile()` hook; `autoFocus` dropped from project modal name input.
- [18-mobile-touch-mk2.md](18-mobile-touch-mk2.md) — mobile layout met touch reality: dynamic viewport (`100dvh`) so inbox / calendar / modal stop hiding under Safari toolbars; long-press hybrid pattern extended to time blocks and rail tasks (220 ms hold, 8 px scroll-cancel, vibrate on lock, `data-pressing` scale+shadow visual fired *only* at lock so quick taps don't flash); `:hover` styles wrapped in `@media (hover: hover)` so tapped buttons don't stay sticky-hovered; document-level `pointerdown` clears `lastBlockId` when the press is outside any `.tblock`; rail restructured so projects collapse inside the same scroll container as task groups; subtask rows in the task modal wrap properly (`flex-start` + `min-width: 0` + `word-break`). **Post-deploy iteration:** PWA install (squared favicon, 32 / 180 / 192 / 512 PNGs, `manifest.webmanifest`, `apple-mobile-web-app-*` meta, `mask-icon`); `env(safe-area-inset-*)` padding on `.app` and the slide-over sidebar so iOS standalone clears the notch; new shared [`useLongPress`](../../web/src/useLongPress.ts) hook (`holdMs` / `cancelPx` / `onContextMenu`) replacing the inbox row's hand-rolled timer; own-user avatar restored on mobile (link button stays desktop-only); totals strip centered; viewport reverted to `initial-scale=1, viewport-fit=cover` after `user-scalable=no` broke Chrome DevTools trackpad scroll; the `font-size: 16px !important` input rule was reverted per user preference.
